## Key Decisions
- Used FastAPI BackgroundTasks for pick embedding (not Celery) to avoid extra broker dependency
- 1.1s sleep between Gemini embedding calls (free tier: 1 RPM)
- Mock embedding ([0.0]*768) when GEMINI_API_KEY absent (dev/CI mode)
- Idempotent pick indexing: (symbol, scan_date, model_type) unique constraint

## Endpoints Added
- GET  /api/v1/ai/memory/search?q=...   — semantic search over picks + conversations
- POST /api/v1/screener/picks/embed-backfill — re-embed all existing JSON pick files

## How to Test
1. Verify DB: psql -d alphaforge -c "\dt"
2. Add GEMINI_API_KEY to backend/.env
3. POST picks to /api/v1/screener/picks  → check logs for "Background embed"
4. GET /api/v1/ai/memory/search?q=stocks+with+high+ADX → returns ranked picks
5. POST /api/v1/ai/chat with session_id in context → reply includes sources[]
