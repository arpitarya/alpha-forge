backend — Implementation Log
============================

2026-04-26 — Portfolio module: brokers, aggregator, routes, tests, docs
-----------------------------------------------------------------------

Goal: aggregate holdings across Zerodha, Zerodha Coin, Groww, Dezerv,
Wint Wealth (CSV exports — only legal/free path) + Angel One SmartAPI
(genuinely free REST). All under one pluggable BrokerSource ABC so
future paid integrations slot in without touching routes or frontend.

New broker stack — backend/app/services/brokers/
------------------------------------------------
- base.py
    BrokerSource ABC + Holding (Pydantic) + AssetClass / SourceKind /
    SourceStatus enums + SourceInfo. Two sub-paths: API → fetch(),
    CSV → parse(stream, filename). Common sync()/ingest_csv()/info()
    lifecycle. In-memory cache, status, last_synced_at, error.

- csv_sources.py
    Five tolerant parsers — ZerodhaCSVSource, ZerodhaCoinCSVSource,
    GrowwCSVSource (auto-detects stocks vs MF via columns/filename),
    DezervCSVSource (asset class column), WintWealthCSVSource (bonds).
    Helpers: _to_float() forgiving on currency symbols + commas + ₹/$,
    _read_csv() utf-8-sig + skips empty rows + trims, _pick() takes
    first non-empty header alias case-insensitively. Tolerance is
    intentional — broker CSVs change column shapes between exports.

- angel_one.py
    SmartAPI client. POST /loginByPassword (clientcode + password +
    TOTP) → jwt → GET /portfolio/v1/getHolding. pyotp lazy-imported
    so the package boots without it. Reads ANGEL_ONE_API_KEY /
    CLIENT_CODE / PASSWORD / TOTP_SECRET from env. Status auto-set to
    READY when all four creds are present at construction.

- aggregator.py
    HoldingsAggregator — stateless read-side: all_holdings(source?),
    totals, allocation (by asset_class, % sums to 100), treemap (uses
    home-rolled _squarify layout — good enough for ≤24 cells), and
    rebalance against DEFAULT_TARGETS (60/15/15/5/3/2 = equity/MF/
    bond/gold/crypto/cash). Suggestions fire above ±5% drift.

- registry.py
    Process-wide SOURCES dict — one BrokerSource instance per slug.

- __init__.py
    Public barrel re-exporting all of the above.

New routes — backend/app/routes/portfolio.py
--------------------------------------------
GET    /portfolio/holdings?source=<slug>   aggregated + totals + alloc
GET    /portfolio/treemap?source=<slug>    pre-computed cell layout
GET    /portfolio/rebalance                drift + suggestions + targets
GET    /portfolio/sources                  list 6 sources + status
GET    /portfolio/sources/{slug}           single source info
POST   /portfolio/sources/{slug}/upload    CSV ingest (multipart)
POST   /portfolio/sources/{slug}/sync      API pull (Angel One only)
POST   /portfolio/sources/{slug}/reset     clear cached holdings
GET    /portfolio/summary                  legacy short shape (kept)

Cross-source rules: 400 if you POST upload to an API source or sync to
a CSV source. Every read payload includes the SEBI disclaimer.

Honest free-tier reality
------------------------
Of the six requested brokers, only Angel One has a genuinely free
REST API (SmartAPI, free with API key + TOTP). The other five offer
no free public API. We chose CSV upload — the only legal + reliable
free path — over reverse-engineered/scraped private endpoints.
