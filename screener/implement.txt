      compute_labels(), compute_labels_for_symbol()
    - Tested: RELIANCE → 487 valid rows, 24 positive (4.9%), 5 NaN tail rows,

    - No lookahead bias: features use data up to day T, labels use T+1 to T+5
    - Output: dataset/output/dataset.parquet + dataset/output/dataset_stats.txt
    - Functions: build_single_stock_dataset(), apply_quality_filters(),

[x] 4.1 — Baseline Rules Strategy (screener/models/baseline_rules.py)
    - Simple rule-based screener for comparison against ML models
    - 3 strategy variants:
      * strict: ALL of RSI<35 + Volume>2×avg + MACD bullish + above SMA50 (AND)
      * relaxed: At least 2 of the 4 rules fire (MIN_2)
      * momentum: Volume surge + MACD bullish + above SMA50 (AND)
    - Functions: rule_rsi_oversold_bounce(), rule_volume_surge(),
      rule_macd_bullish(), rule_price_above_sma50(), apply_rules(),
      evaluate_baseline()
    - Evaluates: signal count, hit rate, avg return, avg win/loss, profit factor
    - Tested (5-stock dataset, 2,340 rows):
      * strict: 0 signals (too strict for test data)
      * relaxed: 766 signals (32.7%), 4.18% hit rate, profit factor 0.83
      * momentum: 26 signals (1.1%), 11.54% hit rate, profit factor 1.39
      * Random baseline: 4.53% positive rate
    - CLI: python screener/models/baseline_rules.py [--strategy strict|relaxed|momentum|all]

[x] 4.2 — LightGBM Classifier (screener/models/train_lightgbm.py)
    - Primary ML model for stock screener
    - LightGBM with walk-forward cross-validation (no random split!)
    - Handles class imbalance via auto-computed scale_pos_weight (21.08x with test data)
    - Handles NaN natively (no imputation needed)
    - Default hyperparams: 500 trees, lr=0.05, depth=6, subsample=0.8,
      colsample=0.8, reg_alpha=0.1, reg_lambda=0.1
    - Auto-drops all-NaN columns (DELIVERY_PCT, DELIVERY_PCT_5D_AVG)
    - Saves model to models/saved/lightgbm/ (model.txt, meta.json,
      feature_importance.csv)
    - Functions: compute_scale_pos_weight(), create_lightgbm_model(),
      train_lightgbm(), save_model(), load_model()
    - Tested (5-stock dataset, 11 folds, 1,105 test rows):
      * AUC-ROC: 0.5674 (avg/fold: 0.4210 ± 0.3449)
      * Top features: DIST_52W_HIGH_PCT, ATR_14_PCT, DIST_52W_LOW_PCT
      * Note: Low signal count expected with only 5 stocks and 4.5% positive rate
    - CLI: python screener/models/train_lightgbm.py [--no-save] [--threshold N]
           [--n-estimators N] [--learning-rate N]
    - Requires: lightgbm, libomp (brew install libomp on macOS)

[x] 4.3 — XGBoost Classifier (screener/models/train_xgboost.py)
    - Comparison model against LightGBM, same features and CV strategy
    - XGBoost with walk-forward CV, scale_pos_weight for imbalance
    - Default hyperparams: 500 trees, lr=0.05, depth=6, tree_method=hist,
      subsample=0.8, colsample=0.8, reg_alpha=0.1, reg_lambda=1.0
    - Saves model to models/saved/xgboost/ (model.json, meta.json,
      feature_importance.csv)
    - Functions: compute_scale_pos_weight(), create_xgboost_model(),
      train_xgboost(), save_model(), load_model()
    - Tested (5-stock dataset, 11 folds, 1,105 test rows):
      * AUC-ROC: 0.5536 (avg/fold: 0.4235 ± 0.3599)
      * Top features: SMA_50_200_CROSS, OBV, SQUEEZE_SIGNAL
    - CLI: python screener/models/train_xgboost.py [--no-save] [--threshold N]
    - Requires: xgboost

[x] 4.4 — Walk-Forward Cross-Validation (screener/models/walk_forward.py)
    - CRITICAL: Always train on past, predict future — never random split
    - Expanding window: min 12 months training, 1 month test per fold
    - Purge gap: 5 trading days between train end and test start to prevent
      label lookahead (configurable via purge_days)
    - WalkForwardSplit class: generates train/test index splits from DatetimeIndex
    - FoldResult dataclass: stores per-fold classification + trading metrics
    - Computes: accuracy, precision, recall, F1, AUC-ROC, hit rate, avg return,
      avg win/loss, profit factor per fold + aggregated
    - Utility functions: get_feature_columns(), drop_all_nan_columns()
    - Tested: 11 folds generated from 2024-05-08 to 2026-03-24
      * Fold 1: Train 1,220 rows → Test 110 rows
      * Fold 11: Train 2,265 rows → Test 60 rows
    - CLI demo: python screener/models/walk_forward.py [--min-train-months N]
    - Requires: scikit-learn

[x] 4.5 — Feature Importance & Selection (screener/models/feature_importance.py)
    - Compares feature importance across LightGBM and XGBoost
    - Ranks features by average rank across models

    - Feature selection: drops features below min importance threshold
    - Functions: get_model_importance(), compare_model_importances(),

[x] 5.1 — Backtest Engine (screener/backtest/engine.py)
    - BacktestEngine class: walk-forward backtest simulator
    - For each test period: ranks stocks by predicted probability, selects top N
    - Simulates: buy at next day's open, sell after 5 trading days at close
    - Configurable top N (default: 5, 10, 20 picks per signal date)
    - Two modes: quick (uses FORWARD_RETURN_5D) and realistic (loads OHLCV data
      for actual open/close prices via use_ohlcv_prices flag)
    - Methods: run_from_predictions(), run_from_walk_forward(), run_baseline()
    - Convenience: run_full_backtest() runs all models + baselines at all top-N configs
    - Equity curve built from period-weighted returns
    - Tested: 8 strategy configurations (2 models × 2 top-N + 2 baselines × 2 top-N)
    - CLI: python screener/backtest/engine.py [--models lgb xgb] [--top-ns 5 10 20]

[x] 5.2 — Indian Market Cost Model (embedded in engine.py)
    - CostModel dataclass with all Indian delivery trade costs:
      * Brokerage: ₹20/order flat (or 0.03% for percentage brokers)
      * STT: 0.1% on sell side (delivery trades)
      * Exchange transaction charges: 0.00345% on both sides
      * GST: 18% on brokerage + exchange txn charges
      * Stamp duty: 0.015% on buy side
    - compute_costs(): detailed per-component cost breakdown
    - compute_cost_pct(): approximate round-trip cost as percentage
    - Tested: ₹1L trade → total round-trip cost ₹173.46 (0.17%)
    - Supports both flat (discount broker) and percentage (full-service) brokerage modes

[x] 5.3 — Performance Metrics (screener/backtest/metrics.py)
    - Trade-level metrics: hit rate, avg win/loss, profit factor, win/loss ratio,
      expectancy, best/worst trade, total costs, gross profit/loss
    - Portfolio-level metrics: Sharpe ratio (annualized, 6.5% risk-free for India),
      Sortino ratio, CAGR, max drawdown, Calmar ratio, total return
    - Combined: compute_all_metrics() returns both trade + portfolio metrics
    - Benchmark checker: compares metrics against target benchmarks from PLAN.md
      (hit >55%, avg win >5%, avg loss <3%, PF >1.5, Sharpe >1.5, MaxDD <20%,
       CAGR >25%, win/loss >1.5)
    - format_metrics_table(): human-readable formatted output
    - Tested: synthetic data → all metric calculations verified correct

[x] 5.4 — Comparison Report (screener/backtest/report.py)
    - Side-by-side comparison: Rules baselines vs LightGBM vs XGBoost
    - generate_comparison_table(): DataFrame with one row per strategy, sorted by PF
    - generate_text_report(): full formatted report with summary table + detailed
      per-strategy metrics + benchmark scores + best-strategy-by-metric section
    - save_report(): saves report to reports/backtest_report.txt + CSV
    - save_trades_csv(): exports individual trade lists per strategy
    - Output: reports/backtest_report.txt (14KB) + reports/backtest_report.csv
    - Tested (5-stock dataset, 8 strategies):
      * Best by PF: baseline_momentum_top5 (PF=1.25, 26 trades, 46.2% hit rate)
      * ML models performed poorly due to small universe (5 stocks, 4.5% pos rate)

================================================================================
KNOWN ISSUES / NOTES
================================================================================

- nselib 2.4.x does not expose fii_dii_trading_activity — need jugaad-data fallback
  or newer nselib version
- price_volume_and_deliverable_position_data requires symbol param in nselib 2.4.x;
  bhav_copy_with_delivery used as fallback for market-wide delivery data
- yfinance TzCache warning is cosmetic (non-blocking), appears on macOS
- Full OHLCV download for ~2000 stocks takes ~10-15 minutes (4-5 batches of 500)
- Survivorship bias acknowledged: yfinance may miss delisted stocks
- Delivery % features have low coverage (~1-3/492 rows for RELIANCE) because
  supplementary data is only fetched for recent date range (7-30 days).
  Need to fetch historical delivery data across 2yr for full coverage.
- yfinance index parquet files have MultiIndex columns — handled in relative_strength.py
  and build_features.py with column flattening
- nselib date parsing triggers pandas UserWarning about format inference — functional
  but could add explicit format strings for cleanliness
- LightGBM on macOS requires libomp: `brew install libomp`
- Model metrics with 5-stock test dataset (2,340 rows) are not representative of
  production performance. AUC and signal counts will improve significantly with
  the full ~2000-stock universe (~200k+ rows). Models are conservative with small
  datasets due to heavy class imbalance (4.5% positive rate).
- DELIVERY_PCT and DELIVERY_PCT_5D_AVG are 100% NaN in test dataset — auto-dropped
  by all model training pipelines
- Walk-forward CV generates 11 folds with ~2yr data (12-month min training).
  More folds will be available with the full dataset.
- Backtest on 5-stock dataset: momentum baseline outperforms ML models (PF=1.25 vs
  PF=0.59). Expected with tiny universe — ML models need more data to learn patterns.
- Indian market cost model: ~0.17% round-trip with ₹20 flat brokerage (discount broker).
  Increases to ~0.25-0.30% with percentage-based brokers.
- SHAP explanations use TreeExplainer for both LightGBM and XGBoost — fast and exact.
- Live scan produces "no signal" (prob < 0.5 threshold) for test stocks — expected with
  test-trained model. Full universe training will produce actionable signals.
