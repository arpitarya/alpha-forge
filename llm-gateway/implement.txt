- Decision: All providers use single `openai` SDK with `base_url` override

## Phase 2: Provider Implementations ✅
- Created `src/alphaforge_llm_gateway/providers/__init__.py` (barrel export all 5 providers)
- Created `providers/gemini.py` — Google Gemini free tier via generativelanguage endpoint
  - Models: gemini-2.5-flash (default), gemini-2.5-pro, gemini-2.5-flash-lite, gemma-4
  - Rate limits: 15 RPM, 1500 RPD
- Created `providers/groq.py` — Groq free tier (fastest inference)
  - Models: llama-3.3-70b-versatile (default), llama-4-scout, qwen3-32b, kimi-k2
  - Rate limits: 30 RPM, 1000 RPD
- Created `providers/huggingface.py` — HuggingFace serverless inference
  - Models: Qwen/Qwen2.5-72B-Instruct (default)
  - Rate limits: 10 RPM, 1000 RPD
- Created `providers/openrouter.py` — OpenRouter with strict free-model-only enforcement
  - Fetches /api/v1/models, hard-filters to $0 prompt AND completion cost
  - Caches free model list, refreshes every 24 hours
  - Raises CostGuardError for non-free models
- Created `providers/ollama.py` — Local Ollama (private, unlimited)
  - Auto-detects available models via /api/tags
  - Graceful failure if Ollama not running

## Phase 3: Rate Limiting & Cost Guardrails ✅
- Created `rate_limiter.py` — In-memory per-provider token bucket
  - Pre-configured free-tier limits for all 5 providers
  - Auto-resets minute/day counters
  - `acquire()`, `remaining()`, `utilization_pct()`, `update_limits()`
- Created `cost_guard.py` — Zero-cost enforcement ($0 wall)
  - Per-provider free model validation (hardcoded for Gemini/Groq, dynamic for OpenRouter)
  - Warns at >80% rate limit utilization
  - Raises CostGuardError if request would incur cost

## Phase 4: Smart Router & Prompts ✅
- Created `router.py` — QueryType → ranked (provider, model) fallback chains
  - SCREENER_ANALYSIS: Gemini Flash → Groq Llama → Ollama
  - STOCK_EXPLANATION: Gemini Pro → Groq Qwen → HuggingFace
  - CHAT: Groq Llama → Gemini Flash → Ollama
  - SENTIMENT: Gemini Flash-Lite → Groq Scout → HuggingFace
  - TECHNICAL_SUMMARY: Gemini Flash → Groq Qwen → Ollama
  - Routing table is updatable (for benchmark results)
- Created `prompts.py` — Domain-specific prompt templates
  - SCREENER_SYSTEM_PROMPT, EXPLAIN_SYSTEM_PROMPT, STOCK_ANALYSIS_PROMPT, SENTIMENT_PROMPT, TECHNICAL_SUMMARY_PROMPT
  - All include SEBI disclaimer
  - SYSTEM_PROMPTS dict maps QueryType → prompt

## Phase 5: Gateway Class & Public API ✅
- Created `gateway.py` — Main LLMGateway class
  - `__init__()` auto-registers providers based on which API keys are provided
  - `from_env()` reads env vars (GEMINI_API_KEY, GROQ_API_KEY, etc.)
  - `complete()` delegates to router with automatic fallback on rate-limit or error
  - `analyze_screener()` / `explain_picks()` convenience methods with system prompts
  - `providers_status()` health + remaining quota per provider
  - Minimal .env loader (no external dependency)
  - If no cloud keys provided, Ollama is always tried as fallback

## Phase 6: CLI Interface ✅

## Key Decisions
- All 5 providers use single openai SDK with base_url override (no provider-specific SDKs)
- In-memory rate limiter (sufficient for personal use)
- Hard $0 cost wall — CostGuardError if any non-free model requested
- OpenRouter dynamically fetches and caches free model list
- Ollama always included as local fallback when no cloud keys configured
- CLI uses argparse (stdlib, zero extra deps)
- Notebook uses await directly (Jupyter native async)
- Backend wrapper follows same pattern as logging.py wrapping alphaforge-logger
